虽然在方法体内部为参数重新赋值从技术上是可行的，但是最好使用一个临时变量保存中间结果。 这个规则主要是为了检测构造方法参数被分配给自己，而不是同名类变量的情况。比如，忘记使用this。 允许重新为参数赋值，也会降低代码可读性，因为开发者无法知道整个方法调用中使用的是原始参数还是一些临时变量。 而且，一些开发者也希望方法参数的复制对调用一方是可见的，这并不重要，但是可能会产生迷惑。 所以参数都应该当做final。

不符合规则的代码示例

class MyClass {
  public String name;

  public MyClass(String name) {
    name = name;                    // Noncompliant - useless identity assignment
  }

  public int add(int a, int b) {
    a = a + b;                      // Noncompliant

    /* additional logic */

    return a;                       // Seems like the parameter is returned as is, what is the point?
  }

  public static void main(String[] args) {
    MyClass foo = new MyClass();
    int a = 40;
    int b = 2;
    foo.add(a, b);                  // Variable "a" will still hold 40 after this call
  }
}

符合规则的方案

class MyClass {
  public String name;

  public MyClass(String name) {
    this.name = name;               // Compliant
  }

  public int add(int a, int b) {
    return a + b;                   // Compliant
  }

  public static void main(String[] args) {
    MyClass foo = new MyClass();
    int a = 40;
    int b = 2;
    foo.add(a, b);
  }
}
