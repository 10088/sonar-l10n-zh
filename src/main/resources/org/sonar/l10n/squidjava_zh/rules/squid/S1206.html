根据Java语言规范，equals(Object) 和 hashCode()之间存在协议关联：

如果两个类调用equals(Object)方法是相同的，调用每个对象的hashCode应该返回相同的整数值。 调用equals(java.lang.Object)方法是相同的两个类，调用hashCode的结果不一定不同。 然而，开发者应该意识到，对于非相同的两个对方返回不同的hashcode会提升hashtable的性能。

为了符合这个协议，这些方法应该同时继承，或同时重载。

不符合规范的代码示例

class MyClass {    // Non-Compliant - should also override "hashCode()"

  @Override
  public boolean equals(Object obj) {
    /* ... */
  }

}

符合规范的方案

class MyClass {    // Compliant

  @Override
  public boolean equals(Object obj) {
    /* ... */
  }

  @Override
  public int hashCode() {
    /* ... */
  }

}
