无论使用什么日志框架（logback, log4j, commons-logging, java.util.logging, ...），logger应该满足如下条件：

  • private：不要在上级class外边访问。如果其他类需要记录日志，它应该初始化自己的logger。
  • static：不要依赖类的一个实例（对象）。记录日志时，消息可以提供上下文的信息，但是logger应该在class级创建，避免每个对象创建一个logger。
  • final：每个类只创建一次。

不符合规则的代码示例

public class Foo {
  public Logger logger = LoggerFactory.getLogger(Foo.class);         // Noncompliant

  public void doSomething() {
    /* ... */
  }
}

public class Bar {
  public static void main(String[] args) {
    Foo foo = new Foo();
    foo.logger.info("Program started");
    foo.doSomething();
  }
}

符合规则的方案

public class Foo {
  private static Logger LOGGER = LoggerFactory.getLogger(Foo.class); // Compliant

  public void doSomething() {
    /* ... */
  }
}

public class Bar {
  private static Log LOGGER = LogFactory.getLogger(Bar.class);       // Compliant

  public static void main(String[] args) {
    LOGGER.info("Program started");
    Foo foo = new Foo();
    foo.run();
  }
}
