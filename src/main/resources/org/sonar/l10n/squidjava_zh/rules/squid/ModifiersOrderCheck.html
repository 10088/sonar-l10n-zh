Java语言规范推荐使用如下顺序排列修饰符：

  • Annotations
  • public
  • protected
  • private
  • abstract
  • static
  • final
  • transient
  • volatile
  • synchronized
  • native
  • strictfp

不遵守这个规则不会产生技术问题，但是会降低代码的可读性，因为大多数开发者使用标准顺序。

如下代码所示： 

static public void main(String[] args) {   // Non-Compliant
}

应该重构为： 

public static void main(String[] args) {   // Compliant
}
